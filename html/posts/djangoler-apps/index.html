Djangoler Apps

2020-03-01

This the Djangoler project. Inspired by the 12 startups in 12 months by Peter Levels, and other indie hackers approach to build MVPs to test products that can solve problems and perhaps turn into profitable businesses.

Avoiding “shiny” tech.⌗

I am an engineer, currently working in a medium sized company, where I work with Go, using containers, deploying to kubernetes clusters, and with dedicated frontend, backend and systems engineers.

However, for this project I want to use a software stack that allows me to build web apps productively. I will use:

  • Django: for the backend development (also for rendering some pages from templates)
  • Bootstrap 3 and jQuery: even those are obsolete technologies, they fit my abilities as a frontend programmer.
  • Ansible: in order to deploy my MVPs to simple VM, that will run everything required. Don’t want to lose time in infrastructure stuff. (I will be happy if at some point something doesn’t fit modern VM’s capacities).

Also, since I am currently employed, my dedication to the project will be a couple of hours daily (perhaps a little bit more on weekends), because I want to avoid any burnout.

Learning how to promote and marketing the MVPs⌗

In the past I’ve already tried to create some products, and I used the classic “engineer approach” where I kept building features, without any validation about who would be interested int them. Once finished I struggled to find someone that would pay for them.

With the Djangoler apps I want to address that: build small reusable MVPs, get feedback about them, and modify them to fit any needs or just abandon them in the wild.

From scratch⌗

The idea is to start from a clean slate. New accounts everywhere: email, twitter, github, and so one. And keep the Djangoler project separate from all my previous background.

But with code reuse…⌗

Some of the things that want to build are ideas that I have already written down, for some of them I even bought the domains, and for some others ideas I have libraries that I can reuse.

Also I am going to use the same UI / Bootstrap theme everywhere, and if at some point any of them deserves an unique design (people starts liking it) I will update that.

Weekly updates⌗

Since I do not have a lot of time, I will just try to write an update every week.
