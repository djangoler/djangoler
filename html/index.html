This the Djangoler project. Inspired by the 12 startups in 12 months by Peter Levels, and other indie hackers approach to build MVPs to test products that can solve problems and perhaps turn into profitable businesses. Avoiding “shiny” tech. I am an engineer, currently working in a medium sized company, where I work with Go, using containers, deploying to kubernetes clusters, and with dedicated frontend, backend and systems engineers. However, for this project I want to use a software stack that allows me to build web apps productively.
